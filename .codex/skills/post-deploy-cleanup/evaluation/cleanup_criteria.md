# Post Deploy Cleanup QC基準

## 削除前チェック

### デプロイ確認
- [ ] GitHubリポジトリURLが有効
- [ ] リポジトリにファイルが存在する
- [ ] main/masterブランチにpush済み

### 対象確認
- [ ] 削除対象ディレクトリが明確
- [ ] 削除対象に誤りがない
- [ ] 他のエージェントに影響しない

### ユーザー確認
- [ ] 削除対象をユーザーに提示した
- [ ] ユーザーから明示的な許可を得た

## 削除後チェック

### 削除完了
- [ ] `output/{agent_name}/` が存在しない
- [ ] 関連する `Flow/` ディレクトリが存在しない

### 影響確認
- [ ] 他の `output/` 内エージェントに影響なし
- [ ] 他の `Flow/` 内作業ファイルに影響なし
- [ ] テンプレート側ファイルに影響なし

## 判定基準

| 状態 | 判定 |
|------|------|
| 全チェックOK | 完了 |
| デプロイ未確認 | 中止（デプロイ確認後に再実行） |
| ユーザー未確認 | 中止（確認後に再実行） |
| 誤削除リスク | 中止（対象を再確認） |
