# Agent Discovery 評価指標

## 必須チェック項目

### 構造チェック（Pass/Fail）
| 項目 | 基準 | 重要度 |
|------|------|--------|
| ドメイン名 | スネークケースで設定 | Critical |
| 課題記載 | 解決すべき課題が明記 | Critical |
| 対象ユーザー | 利用者が特定されている | Critical |
| 必須機能 | 3つ以上の機能が列挙 | Critical |
| 成功基準 | KPIまたは成功条件が記載 | High |
| フレームワーク調査 | framework_research.mdが存在 | High |
| 承認確認 | 構造承認が記録されている | Critical |

### 内容チェック（スコアリング）
| 観点 | 評価基準 | 配点 |
|------|----------|------|
| 完全性 | 質問項目の80%以上に回答 | 25 |
| 具体性 | 曖昧表現（〜など、〜的な）が5箇所未満 | 25 |
| 整合性 | 課題と機能の対応が明確 | 25 |
| 出典明記 | 業界標準調査に出典・日時あり | 25 |

### 許容例外
- 未決定事項はTODO形式で明記されていれば可
- 機密情報は「非公開」と記載可
- 将来機能は空でも可

## 採点基準
- **Pass**: 全CriticalがPass かつ スコア80点以上
- **Conditional Pass**: 全CriticalがPass かつ スコア60-79点
- **Fail**: CriticalにFailあり または スコア60点未満

## QC報告フォーマット
```
### QC結果: [Pass/Conditional Pass/Fail]
スコア: [XX]/100

#### 指摘事項
1. [項目名]: [指摘内容] (重要度: Critical/High/Medium/Low)

#### 修正推奨
- [具体的な修正アクション]
```
