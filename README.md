# Agent Template - ドメイン特化エージェント生成フレームワーク

高品質な業務特化エージェントを生成するテンプレートベースフレームワークです。

## 🎯 設計思想

- **業界標準準拠**: BABOK、AMA Marketing BOKなど、確立されたフレームワークベース
- **実務即応性**: 実際に使用可能な詳細ワークフロー
- **手作り品質**: 専門知識に基づく高品質テンプレート

## 🚀 クイックスタート

### 明確なエージェントがある場合

```bash
python scripts/enhanced_generate_agent.py --agent-name "Movie" --domain "movie" --description "映画調査・レビュー・サジェストエージェント"
```

### 生成後の必須作業

1. `output/{agent_name}_agent/.cursor/rules/` に移動
2. `agent_paths.mdc` を `{domain}_paths.mdc` にリネーム
3. `template_paths.mdc` を削除
4. `00_master_rules.mdc` を初期化・カスタマイズ
5. 01〜15番の機能ルールを作成

## 📁 生成されるエージェント構造

```
{domain}_agent/
├── Flow/                      # 作業中ドラフト
│   ├── Public/               # 公開可能な作業ファイル
│   └── Private/              # 非公開作業ファイル
├── Stock/                     # 確定版ドキュメント
│   └── programs/             # プログラム/プロジェクト構造
├── Archived/                  # アーカイブ
├── .cursor/                   # Cursor IDE設定
│   ├── rules/                # 実行用ルール(.mdc)
│   └── templates/            # ドキュメントテンプレート
├── scripts/                   # 自動化スクリプト
└── README.md                  # エージェント専用説明書
```

## 🔧 ワークフロー

### 3フェーズアプローチ

1. **Discovery**: ヒアリングと要件定義を実施し、合意内容をドキュメント化
2. **Build**: スクリプトとテンプレートを用いてルール・パス・ドラフトを整備
3. **Feedback**: 評価レポートと改善タスクを更新し、必要に応じてルールを再調整

### 4段階開発プロセス

1. **Phase 1: 設計準備** - ドメイン分析・要件定義・機能設計を整理
2. **Phase 2: ルール実装** - 00_master_rules.mdc と機能ルールを作成
3. **Phase 3: 品質保証** - 実行テストと評価レポートで採点
4. **Phase 4: 継続改善** - レビュー専用ルールで改善サイクルを運用

## 🏗️ アーキテクチャ

### ルール番号体系

- **00**: Master Control Rules（ドメイン固有マスタールール）
- **01-89**: Domain Specialized Rules（各ドメインエリアのルール）
- **90**: Task Management（標準タスク管理）
- **97**: Flow to Stock Rules（ドキュメント確定・移行）
- **98**: Flow Assist（ブレインストーミング支援）
- **99**: Rule Maintenance（システム管理）

## 📋 実装必須ポイント

### トリガー構成の基本ステップ

1. **start_message**: 開始メッセージを提示
2. **collect_existing_info**: 既存情報を収集し最新状況を把握
3. **ask_questions**: 対象セクションの質問を実行
4. **create_output**: テンプレートに基づき成果物ファイルを作成
5. **completion_message**: 完了メッセージを通知

### 使用可能なアクション

- `message`: ユーザーに通知や案内を出す
- `call`: 別ファイルの質問・テンプレートを呼び出す
- `create_markdown_file`: テンプレートを適用してMarkdownを生成
- `gather_existing_info`: 既存情報を収集し、作業の文脈を整理
- `execute_shell`: 外部コマンドを実行し、環境やファイルを整備

## 📊 品質評価フレームワーク

### 4つの評価観点

1. **記述規則準拠性**: YAML構文・ルール番号体系・ファイル構造への準拠
2. **機能充足性**: 業界標準準拠・ワークフロー網羅・機能間連携・実務適用性
3. **ゴール適合性**: 初期要件適合・ターゲットユーザー適合・効率化効果・学習曲線
4. **実用性**: 質問品質・テンプレート品質・エラーハンドリング・拡張性

### ブラッシュアップ基準

- **必須**: いずれかの観点が3.5未満
- **推奨**: 全観点3.5以上だが4.0未満
- **不要**: 全観点4.0以上

## 🔍 ドメイン例

### BABOK（ビジネスアナリシス）
- プロジェクト初期化・体制構築
- 要求抽出・検証
- ビジネスケース・ステークホルダー分析
- ソリューション評価・データモデリング
- 変更戦略・最終評価

### マーケティング
- 市場調査・競合分析・トレンド分析
- ターゲット設定・ブランド戦略・価格戦略
- キャンペーン企画・コンテンツ企画・チャネル戦略
- 効果測定・分析・最適化

### 法務
- 契約書レビュー・作成・修正
- コンプライアンス確認・リスク評価・紛争分析
- 社内規程策定・法規制対応・研修資料作成

## ⚠️ 重要な注意事項

- 必ず業界標準フレームワークに基づくこと
- 機械的な生成ではなく、手作りの品質を維持すること
- 生成後の手動修正は必須（自動生成は骨格のみ）
- 品質評価・ブラッシュアップは必須プロセス
- 実際の業務経験者による検証を推奨
