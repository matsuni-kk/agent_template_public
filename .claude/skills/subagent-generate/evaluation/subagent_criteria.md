# サブエージェント生成 QC基準

## サブエージェント化判定チェック

### 判定の妥当性
- [ ] コンテキスト不要で独立実行可能か確認されている
- [ ] 単なる評価軸チェックでないことが確認されている
- [ ] 並列実行のメリットがあることが説明されている
- [ ] 携帯Skillsが必要な理由が明確

### 不適切なサブエージェント化の検出
- [ ] 親コンテキスト必須のタスクがサブエージェント化されていない
- [ ] evaluation/*.mdで十分なチェックがagents/*.mdになっていない
- [ ] オーバーヘッドが大きい軽微なタスクが含まれていない

## 携帯Skills設定チェック

### 適切性
- [ ] サブエージェントのタスクに関連するSkillsが選定されている
- [ ] 携帯Skillsだけでタスク完了可能
- [ ] 不要なSkillsが含まれていない

### 参照ファイル
- [ ] 各携帯Skillの参照すべきファイルが明記されている
- [ ] SKILL.md, assets/, evaluation/の使い分けが適切

## 入出力定義チェック

### 入力定義
- [ ] 必須パラメータが明確
- [ ] 親コンテキストに依存しない設計
- [ ] スコープが明確に定義されている

### 出力定義
- [ ] 構造化フォーマットが定義されている
- [ ] アクショナブルな出力
- [ ] 優先度・重要度の付与ルールがある

## タイプ別チェック

### research-agent
- [ ] Web検索クエリが定義されている
- [ ] 情報ソースの優先度がある
- [ ] 取得日・ソースの記録ルールがある

### feedback-agent
- [ ] レビュー観点が明確
- [ ] 重み付けがある
- [ ] 改善提案のフォーマットがある

### ideation-agent
- [ ] 最低アイデア数が指定されている
- [ ] Pros/Cons比較のルールがある
- [ ] 推奨案選定の基準がある

### validation-agent
- [ ] 検証項目リストがある
- [ ] 判定基準が明確
- [ ] リスク評価のルールがある

### parallel-check-agent
- [ ] 並列実行の対象が明確
- [ ] ファイルパターンが指定されている
- [ ] 集約ルールがある

## 判定基準

| 状態 | 判定 | アクション |
|------|------|-----------|
| 全項目OK | 承認 | 生成完了 |
| 携帯Skills不適切 | 条件付き承認 | Skill選定を修正 |
| サブエージェント化判定NG | 差し戻し | 再判定必要 |
| 入出力定義不明確 | 条件付き承認 | 定義を補完 |
