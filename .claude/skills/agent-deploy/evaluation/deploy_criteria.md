# Agent Deploy QC基準

## 必須ファイル存在チェック

### 最低限必須
- [ ] CLAUDE.md が存在し、内容が適切
- [ ] README.md が存在し、エージェント概要が記載
- [ ] .claude/skills/ に少なくとも1つのSkillが存在
- [ ] 各SkillにSKILL.mdが存在

### 推奨ファイル
- [ ] scripts/update_agent_master.py が存在
- [ ] scripts/validate_skills.py が存在
- [ ] .gitignore が存在

## マルチプラットフォーム同期チェック

### ファイル同期
- [ ] AGENTS.md の内容が CLAUDE.md と一致
- [ ] .github/copilot-instructions.md の内容が一致
- [ ] .gemini/GEMINI.md の内容が一致
- [ ] .kiro/steering/KIRO.md の内容が一致

### 同期タイムスタンプ
- [ ] 全ファイルの更新日時が近い（同期済みの証拠）

## セキュリティチェック

### 機密情報
- [ ] .env ファイルがリポジトリに含まれていない
- [ ] APIキー、トークンがハードコードされていない
- [ ] credentials.json 等がコミットされていない

### .gitignore
- [ ] .env が除外対象に含まれている
- [ ] 機密ファイルパターンが除外されている

## リポジトリ作成チェック

### 設定
- [ ] プライベートリポジトリとして作成された
- [ ] リポジトリ名が適切（スネークケース）
- [ ] 初期コミットが成功している

### プッシュ
- [ ] リモートへのプッシュが成功している
- [ ] リポジトリURLが取得できる

## 判定基準
- 必須ファイル全て存在 + セキュリティOK: 承認
- 推奨ファイル不足: 条件付き承認（警告を記録）
- 機密情報漏洩リスク: 差し戻し（即時対応必要）
