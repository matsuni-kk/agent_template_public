# Evaluation スケルトンテンプレート

## evaluation/{skill}_criteria.md テンプレート

```markdown
# {Skill名} 評価指標

## 必須チェック項目

### 構造チェック（Pass/Fail）
| 項目 | 基準 | 重要度 |
|------|------|--------|
| {check_1} | {criteria_1} | Critical |
| {check_2} | {criteria_2} | Critical |
| {check_3} | {criteria_3} | High |
| {check_4} | {criteria_4} | Medium |

### 内容チェック（スコアリング）
| 観点 | 評価基準 | 配点 |
|------|----------|------|
| 完全性 | {completeness_criteria} | 30 |
| 正確性 | {accuracy_criteria} | 30 |
| 品質 | {quality_criteria} | 20 |
| 形式 | {format_criteria} | 20 |

### 許容例外
- {exception_1}
- {exception_2}

## 採点基準
- **Pass**: 全CriticalがPass かつ スコア80点以上
- **Conditional Pass**: 全CriticalがPass かつ スコア60-79点
- **Fail**: CriticalにFailあり または スコア60点未満

## QC報告フォーマット
```
### QC結果: [Pass/Conditional Pass/Fail]
スコア: [XX]/100

#### 指摘事項
1. [項目名]: [指摘内容] (重要度: Critical/High/Medium/Low)

#### 修正推奨
- [具体的な修正アクション]
```
```

## ドメイン別評価観点例

### 会計（Accounting）
| 観点 | 評価基準 |
|------|----------|
| GAAP準拠 | 会計基準に準拠した記載 |
| 勘定科目正確性 | 正しい勘定科目の使用 |
| 借貸一致 | 借方・貸方の一致 |
| 日付正確性 | 正しい会計期間の記載 |

### マーケティング（Marketing）
| 観点 | 評価基準 |
|------|----------|
| ターゲット明確性 | ペルソナの具体性 |
| データ根拠 | 市場データに基づく分析 |
| KPI設定 | 測定可能な指標 |
| 競合差別化 | 独自性の明示 |

### 法務（Legal）
| 観点 | 評価基準 |
|------|----------|
| 法的正確性 | 適用法令の正確な引用 |
| リスク網羅性 | リスク項目の漏れなし |
| 条項整合性 | 契約条項間の矛盾なし |
| 用語統一性 | 法的用語の一貫性 |

### 人事（HR）
| 観点 | 評価基準 |
|------|----------|
| 公平性 | 評価基準の客観性 |
| 法令遵守 | 労働法規への準拠 |
| 記録完全性 | 必須項目の網羅 |
| 個人情報保護 | プライバシー配慮 |

### プロジェクト管理（PMBOK）
| 観点 | 評価基準 |
|------|----------|
| スコープ明確性 | 成果物定義の明確さ |
| スケジュール妥当性 | 工程見積りの現実性 |
| リスク識別 | リスク項目の網羅 |
| ステークホルダー | 関係者の特定 |

## 重要度定義

| 重要度 | 定義 | 対応 |
|--------|------|------|
| Critical | 欠如すると成果物として成立しない | 必須修正 |
| High | 品質に重大な影響 | 強く推奨修正 |
| Medium | 品質向上に寄与 | 推奨修正 |
| Low | あれば望ましい | 任意修正 |

## QCサイクル

```
1. Subagent が evaluation/{criteria}.md を読む
2. 成果物を評価基準に照らして検査
3. QC報告フォーマットで指摘
4. 指摘を最小差分で反映（最大3回）
5. 反映結果を成果物に記録
```
